# 웹 호스팅 플랫폼 상세 비교 분석

이 문서는 `tech-blog` 구축을 위한 세 가지 주요 정적 호스팅 플랫폼(GitHub Pages, Vercel, Netlify)의 장단점, 특징, 그리고 기술적 차이점을 상세하게 비교합니다.

## 1. GitHub Pages (The Native Choice)
GitHub 저장소에 있는 코드를 바로 웹사이트로 만들어주는 가장 기본적인 서비스입니다.

### 특징
- **완전 통합**: 별도의 서비스 가입 없이 GitHub 설정 메뉴에서 바로 활성화 가능합니다.
- **정적 사이트 전용**: HTML, CSS, JavaScript 파일만 호스팅 가능합니다. (PHP, Python, Node.js 서버 코드 실행 불가)
- **Jekyll 엔진**: Ruby 기반의 정적 사이트 생성기인 Jekyll이 내장되어 있어, 마크다운 파일을 자동으로 HTML로 변환해줍니다.

### 장점 (Pros)
- **접근성**: GitHub 사용자라면 누구나 즉시 사용 가능. 러닝 커브가 거의 없습니다.
- **비용**: Public 리포지토리에 대해서는 대역폭 제한 없이 **완전 무료**입니다.
- **관리 용이성**: 코드와 배포 설정이 한 곳(GitHub)에 있어 관리가 단순합니다.
- **Custom Domain**: 개인 도메인 연동이 무료로 가능합니다.

### 단점 (Cons)
- **속도**: 글로벌 CDN 기능이 전문 호스팅 업체(Vercel, Netlify)에 비해 상대적으로 단순하여 로딩 속도가 느릴 수 있습니다.
- **기능 제한**: 서버리스 함수(Serverless Functions), 동적 이미지 최적화, 폼 처리 등의 고급 기능을 제공하지 않습니다.
- **Jekyll 종속성**: 다른 프레임워크(React, Vue 등)를 쓸 경우 빌드 과정(GitHub Actions)을 따로 설정해줘야 하는 번거로움이 있을 수 있습니다.

---

## 2. Vercel (The Modern & Performance Choice)
React 기반의 프레임워크인 **Next.js**를 만든 팀에서 운영하는 호스팅 플랫폼으로, 현대적인 웹 개발의 표준으로 자리 잡고 있습니다.

### 특징
- **Zero Configuration**: 대부분의 프레임워크(Next.js, React, Vue, Svelte 등)를 자동으로 감지하고 최적의 설정을 적용합니다.
- **DPR & ISR**: 정적 페이지뿐만 아니라, 필요에 따라 서버에서 페이지를 그려주는(Server-side Rendering) 기능이나 증분 정적 재생성 등 고급 기술을 쉽게 쓸 수 있습니다.
- **Preview Deployments**: 코드를 수정해서 PR(Pull Request)을 날리면, 실제 배포 전에 미리 볼 수 있는 주소를 자동으로 생성해줍니다.

### 장점 (Pros)
- **압도적인 성능**: 전 세계에 퍼져 있는 Edge Network(CDN)를 통해 콘텐츠를 매우 빠르게 전송합니다.
- **개발자 경험(DX)**: GitHub와 연동하면 코드 푸시 -> 자동 빌드 -> 자동 배포 과정이 매우 매끄럽게 진행됩니다.
- **서버리스 함수**: 백엔드 서버 없이도 간단한 API(로그인, DB 연동 등)를 만들 수 있습니다.
- **Next.js 최적화**: Next.js를 사용한다면 무조건 Vercel이 최고의 선택입니다.

### 단점 (Cons)
- **복잡성**: 단순한 HTML 파일 하나 띄우기에는 기능이 너무 많고 과분하게 느껴질 수 있습니다.
- **계정 관리**: GitHub 외에 Vercel이라는 별도 서비스에 가입하고 연동해야 합니다.

---

## 3. Netlify (The Feature-Rich & Stable Choice)
정적 웹사이트 호스팅의 선구자로, 개발자가 아니어도 다루기 쉬운 편의 기능들을 많이 제공합니다.

### 특징
- **Forms**: 별도의 백엔드 코드 없이 HTML 태그에 `netlify` 속성만 추가하면, 사용자 입력을 받아서 저장하고 이메일로 보내주는 기능을 제공합니다.
- **Identity**: 로그인/회원가입 기능을 쉽게 구현할 수 있는 인증 모듈을 제공합니다.
- **Instant Rollbacks**: 배포가 잘못되었을 때, 버튼 클릭 한 번으로 이전 버전으로 즉시 되돌릴 수 있습니다.

### 장점 (Pros)
- **편의 기능**: 폼 처리, A/B 테스팅, 인증 등 웹사이트 운영에 필요한 부가 기능이 매우 잘 되어 있습니다.
- **직관적 UI**: 드래그 앤 드롭으로 폴더를 올려서 배포할 수도 있고, 설정 화면이 매우 직관적입니다.
- **무료 플랜**: 개인 사용자가 쓰기에 무료 플랜의 허용량이 넉넉합니다.

### 단점 (Cons)
- **Next.js 지원**: Next.js의 최신 기능(예: 최신 이미지 최적화, ISR 등) 적용 속도가 Vercel보다는 한 템포 늦을 수 있습니다.

## 💡 요약 및 추천

| 구분 | GitHub Pages | Vercel | Netlify |
| :--- | :--- | :--- | :--- |
| **난이도** | 매우 쉬움 | 쉬움 (자동화 강력) | 쉬움 (UI 친화적) |
| **속도/성능** | 보통 | **최상** | 우수 |
| **동적 기능** | 없음 (정적만 가능) | **강력함** (서버리스, SSR) | 우수 (Forms, Auth) |
| **비용(개인)** | 무료 | 무료 (Hobby) | 무료 (Starter) |
| **주요 용도** | 문서, 간단 블로그 | **고성능 블로그**, 웹앱 | 랜딩 페이지, 포트폴리오 |

**결론:**
- **"나는 코딩보다는 글쓰기에 집중하고 싶고, 설정이 제일 간단한 게 좋다"** 👉 **GitHub Pages**
- **"최신 기술(React, Next.js)을 써서 빠르고 멋진 블로그를 만들고 싶다"** 👉 **Vercel** (강력 추천)
- **"회원가입, 문의하기 폼 같은 기능이 필요하다"** 👉 **Netlify**
