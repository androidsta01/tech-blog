# 기술 블로그 호스팅, 어디서 해야 할까?

별도의 서버 관리 없이 웹페이지를 띄울 수 있는 주요 플랫폼들을 비교해봤습니다.

## 비교해본 플랫폼들

### 1. GitHub Pages

**장점:**
- 완전 무료입니다 (공개 저장소 기준)
- GitHub와 완벽하게 연동됩니다
- 자동 배포가 가능합니다 (GitHub Actions 사용)
- 내 도메인을 연결할 수 있습니다
- HTTPS가 자동으로 적용됩니다

**단점:**
- 정적 사이트만 가능합니다
- 빌드 시간에 제한이 있습니다
- 월 대역폭 제한이 있습니다 (100GB)

**이런 분들께 추천합니다:**
- 이미 GitHub를 사용하고 계신 분
- 마크다운으로 블로그를 쓰고 싶으신 분
- 간단한 정적 사이트를 만들고 싶으신 분

---

### 2. Vercel

**장점:**
- 배포 속도가 정말 빠릅니다
- Next.js에 최적화되어 있습니다
- 서버리스 함수를 지원합니다
- HTTPS가 자동으로 적용됩니다
- 전 세계 어디서나 빠른 속도를 보장합니다 (글로벌 CDN)

**단점:**
- 무료 플랜에 제한이 있습니다 (대역폭, 빌드 시간)
- 설정이 복잡할 수 있습니다

**이런 분들께 추천합니다:**
- Next.js 프로젝트를 진행하시는 분
- 서버리스 API가 필요하신 분
- 빠른 성능이 중요하신 분

---

### 3. Netlify

**장점:**
- 사용하기 정말 쉽습니다
- 폼 처리 기능이 내장되어 있습니다
- 서버리스 함수를 지원합니다
- 자동 배포가 됩니다
- A/B 테스트 기능이 있습니다

**단점:**
- 무료 플랜 대역폭 제한이 있습니다 (100GB/월)
- 빌드 시간에 제한이 있습니다

**이런 분들께 추천합니다:**
- 간단한 정적 사이트를 만드시는 분
- 폼이 필요한 사이트를 만드시는 분
- 자동화된 배포를 원하시는 분

---

## 최종 선택: GitHub Pages

이 프로젝트에서는 **GitHub Pages**를 선택했습니다.

**선택한 이유:**
1. 이미 GitHub를 사용하고 있었습니다
2. 완전 무료입니다
3. GitHub Actions로 자동 배포가 가능합니다
4. 마크다운 기반 블로그에 딱 맞습니다
5. 설정이 정말 간단합니다

**어떻게 구현했나요:**
- `main` 브랜치에 소스 코드를 저장합니다
- GitHub Actions로 빌드 스크립트를 실행합니다
- `gh-pages` 브랜치에 빌드 결과를 배포합니다
- 자동으로 사이트가 업데이트됩니다

## 결론

각 플랫폼마다 장단점이 있지만, 기술 블로그를 만든다면 GitHub Pages가 가장 적합한 선택입니다. 특히 마크다운으로 글을 작성하고 Git으로 버전 관리를 한다면 최고의 조합이라고 생각합니다.
